# Frontend for design eindopdracht

## Door Arnie Schrage

Stel je eens voor, er breekt een enorme pandemie uit waardoor je verplicht thuis moet zitten. Dan zou het zo maar eens kunnen dat je door die absurde hoeveelheid tijd die je aan games besteedt (Jep, jij ook) ineens klaar bent. Je platinum trophies zijn binnen, die ene sidequest is gehaald en je speedrun heeft al eens het wereldrecord verbroken.

"OH NEE, WAT NU?!" hoor ik je al zeggen.

NextPlay helpt jou in deze extreem lastige situatie, je staat er niet alleen voor.

Wanneer je even geen inspiratie hebt en niet weet welke game je moet spelen, kan NextPlay een suggestie geven. Vul de filters in, en voor je het weet ben je alweer aan het grinden voor die achievements! #Covidgaming

## Techniek

De API is van RAWG, een grote videogame database. Deze api wordt aangesproken nadat de gebruiker zijn filters heeft toegepast. Er worden op basis van de ingevulde opties variabelen aangemaakt, die vervolgens worden omgezet naar een URI. De response van de API wordt in een array opgeslagen, die vervolgens gebruikt wordt om de pagina dynamisch in te vullen met de content die is opgevraagd.

## Volgende stappen

Deze tool kan nog veel verbeterd worden. Als ik het nog verder zou uitbreiden, zou ik willen kijken of ik data van meerdere api's zou kunnen samenvoegen, zodat er bijv. automatisch een linkje komt om de game te kopen (Uiteraard waar die het goedkoopst is), ik zou het geheel visueel willen verbeteren, en de rating functie werkend willen krijgen. Ook wil ik nog wat verder verdiepen in het opzetten van queries zodat de filters geavanceerder zijn, en misschien wel realtime in beeld komen. Dit niveau heb ik helaas nog niet bereikt. Ik zou liever de code willen leren optimaliseren, want waarschijnlijk kan het efficienter, maar goed, voor een prototyping vak gaat het nog wel. En gezien ik bij de eerste les nog maar net een Eventlistener kon toepassen, zie ik dat mijn JS niveau een aardige sprong heeft gemaakt!
